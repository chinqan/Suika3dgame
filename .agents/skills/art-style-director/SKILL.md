# 物理落下益智遊戲 — 美術風格與環境敘事總監

**職能**：定義並維護霓虹 Cyberpunk 視覺風格、配色系統、形狀繪製規範、環境敘事設計。

---

## When to Use

- 定義或調整遊戲整體美術風格
- 設計新形狀的視覺外觀
- 修改配色系統（環境色、形狀色）
- 調整 Glow / Blur / Additive Blend 等視覺效果
- 設計環境場景元素（3D 地板、牆壁光效、格線光流）
- 擴展隱性世界觀或能量隱喻設定

---

## 🚨 Critical Rules（不可違反）

### 氛圍設計意圖必須記錄

- **每個視覺元素都必須附帶「氛圍意圖」說明** — 不能只寫參數值，要寫「為什麼這樣設計」
- **alpha / 透明度梯度必須記錄公式** — 不能只寫「有漸變效果」，要寫清楚算法
- **顏色的敘事意義必須定義** — 不能只說「用青色」，要說「青色 = 冷電流 = 數位空間基調」
- **Game Over / 狀態轉變的氛圍變化必須定義** — 環境如何隨遊戲狀態改變

### 環境視覺的完整性規範

- **可見元素**（地板、牆壁、光流）：必須有完整的繪製參數表（顏色、寬度、alpha、座標公式）
- **不可見元素**（背景微格線、次層氛圍）：也必須記錄——它們是「感覺不到但不能沒有」的設計
- **Texture 規格**：程式生成的 Texture 必須定義每一層的半徑、alpha、顏色，確保可復原

### 氛圍設計意圖模板

每個環境場景元素必須使用以下格式記錄：

```markdown
### [元素名稱]

| 屬性 | 值 | 說明 |
|------|-----|------|
| ... | ... | ... |

> **氛圍意圖**：[這個元素對玩家產生什麼感受？為什麼這樣設計？]
```

## 核心風格定義

**風格關鍵詞**：Cyberpunk × Neon Vector × Arcade

| 原則 | 規則 |
|------|------|
| **對比度** | 純黑虛空 vs 高飽和霓虹——沒有中間灰色地帶 |
| **Glow 至上** | 所有物件都在「發光」——線條是能量，不是物質 |
| **運動感** | 粒子、脈衝、光流——畫面中沒有完全靜止的東西 |
| **亮度=重要性** | 越亮 = 越重要。玩家控制的形狀最亮 |
| **環境敘事** | 視覺環境暗示主題，不需文字故事 |

### 風格參考

- **Tron**：3D 透視地板格線
- **Geometry Wars**：霓虹幾何體在深色空間中
- **Vaporwave**：高飽和度漸層色盤
- **Arcade 復古**：Orbitron 科技字型、大寫標題

## 配色系統設計

### 環境配色

| Token | HEX | 用途 |
|-------|-----|------|
| `--color-bg-primary` | `#0a0a0f` | 全域背景（近乎純黑） |
| `--color-bg-secondary` | `#0d0d1a` | 遊戲包裝漸層 |
| `--color-accent` | `#00FFFF` | 牆壁、分數、主按鈕（青色） |
| `--color-danger` | `#FF3131` | Game Over 線、Hard 按鈕 |

### 形狀配色原則

- **從冷到暖再回歸白光** — 暗示能量從基礎到終極的進化
- 每個等級的顏色必須在深黑背景上高對比可辨識
- 鄰近等級不可使用相近色（避免合成目標混淆）
- **Lv.8（最高）= 白色** — 所有光的合一，視覺上最特殊

## 形狀三層繪製架構

每個形狀由 3 層視覺結構組成：

```
Layer 3 (外): Glow 光暈 ── 3 圈遞減透明度外環
Layer 2 (中): Main Shape ── 半透明填充 + 粗邊框 + 內圈裝飾
Layer 1 (內): Bubble      ── 碰撞體可視化圓形覆蓋
```

### 設計規範

| 層 | 元素 | 規則 |
|----|------|------|
| Glow | 3 圈外環 | radius +4/+8/+12px，alpha 遞減 0.08→0.04→0.027 |
| Main | 填充 | 形狀色 alpha=0.08（極淡填充，不可不透明） |
| Main | 邊框 | 形狀色 alpha=1.0，width=6（霓虹管感） |
| Main | 內圈 | 半徑 ×0.55，width=3，alpha=0.2 |
| Bubble | 覆蓋 | 碰撞半徑 +3px，alpha=0.15 |

### 多邊形 vs 圓形

- **sides=0** → `g.circle()`
- **sides>0** → `g.poly(buildPolygonPoints())`
- 頂點起始角 = `-π/2`（第一個頂點朝正上方）

## 環境場景設計

### 3D 透視地板

| 參數 | 值 | 設計意圖 |
|------|-----|---------|
| hLines | 8 | 水平格線，模擬深度 |
| vLines | 10 | 垂直格線，模擬範圍 |
| power | 2.5 | 透視壓縮——越大越有深度感 |

**水平格線 alpha**：`0.5 + 0.5 × (1 - tC)` — 近亮遠暗

> **氛圍意圖**：亮度消退模擬視覺深度，遠端消融於虛空。

**垂直格線 alpha**：`(0.35 + 0.35 × (1 - |frac-0.5|×2)) × 0.9` — 中央亮兩側暗

> **氛圍意圖**：引導視線聚焦容器中央，製造「隧道視覺」效果。

**背景微格線**：白色 alpha=0.025，step=45px

> **氛圍意圖**：潛意識層面強化數位空間感——感覺不到但不能沒有。

### 格線光流

- Light flow 粒子嚴格沿格線移動（lineIdx 對應地板格線）
- 60% 水平 / 40% 垂直——暗示能量流向
- 亮度曲線：中點最亮、兩端漸滅——能量脈衝
- GlowTexture：由外到內 4 層（32/22/14/8px），核心白色 alpha=1.0，外層青色 alpha=0.08
- **繪製順序影響最終效果** — 內層覆蓋外層，必須記錄繪製順序
- Game Over 時停止生成，已有光流自然消散

> **氛圍意圖**：光流 = 能量血液。Game Over 時停止 = 能量斷絕。

## 環境敘事（隱性世界觀）

> 此遊戲為「無文字敘事」——所有故事透過視覺暗示。

### 形狀能量隱喻

每個等級對應不同的「能量維度」：

| 階段 | 隱喻 |
|------|------|
| Lv.0-2 | 基礎能量——不穩定、正在凝聚 |
| Lv.3-5 | 結構化能量——共振、跨越 |
| Lv.6-7 | 宇宙級能量——力場、跨維度 |
| Lv.8 | 終極統一場——圓滿、不可分裂 |

### 設計規則

- 霓虹幾何體 = 數位能量晶體
- 合成 = 同頻共鳴融合
- 容器溢出 = 系統崩潰
- 深黑背景 = 虛空中的數位微宇宙

## 工作流程

1. **風格指南** — 定義關鍵詞、參考、不可違反的視覺原則
2. **配色系統** — 環境色 + 形狀色完整表
3. **形狀繪製** — 三層架構規範
4. **環境元素** — 3D 地板、牆壁、光流
5. **世界觀** — 隱性敘事的能量隱喻
6. **視覺驗證** — 截圖確認對比度、可讀性、風格一致性
