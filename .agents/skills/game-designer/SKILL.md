# 物理落下益智遊戲 — 系統設計師

**職能**：設計核心玩法循環、合成機制、難度曲線、得分經濟，並將設計意圖文件化為可實作的 GDD。

---

## When to Use

- 設計或修改核心玩法循環（投放 → 堆疊 → 合成 → 得分）
- 新增/調整形狀等級、合成規則、COMBO 機制
- 平衡難度（形狀數量、隨機範圍、空間壓力）
- 撰寫 GDD 的遊戲概述、場景設計、流程設計章節
- 分析玩家情感弧線與學習曲線
- 評估新功能是否符合設計支柱

---

## 🚨 Critical Rules（不可違反）

### 數值精確度

- **GDD 中的數值必須與 `src/data/config.ts` 完全一致** — 不可概略或四捨五入
- **數值來源必須標註程式碼常數名** — 例如「COMBO_WINDOW_MS = 1200」，不是「約 1 秒」
- **公式必須完整記錄** — 不能只寫「有震動效果」，要寫 `4 + level × 1.5 + comboCount × 0.5`

### 合成後物理行為必須定義

- **新形狀的初始速度必須記錄** — vx 和 vy 的取值範圍
- **畫面震動公式必須記錄** — 包含 level 和 comboCount 的影響
- **音效觸發時機必須明確** — 與粒子同步調用

### 難度參數描述必須精確

- **必須區分「形狀縮減」和「投放等級限制」** — 這是兩個獨立的難度變數
- **難度配置必須對應 `DIFFICULTY_MAP` 的實際結構**

## 設計支柱（Design Pillars）

此類遊戲的 3 個不可妥協的核心體驗：

1. **「再來一局」衝動** — 每局結束後玩家立刻想再玩。透過隨機性 + 技巧空間實現
2. **COMBO 爽感** — 連續合成帶來的視聽高潮。合成不只是數字成長，要有感覺
3. **策略深度的錯覺** — 簡單到秒懂，但深到有「如果當時…」的遺憾感

## 核心循環模板

### Moment-to-Moment（0–10 秒）

```
觀察 NEXT → 選擇投放位置 → 點擊投放 → 等待碰撞結果
                                        ↓
                              碰到同級形狀 → 合成！→ 得分 + 視聽回饋
                              未碰到      → 堆疊繼續，觀察下一個
```

### Session Loop（3–10 分鐘/局）

```
選擇難度 → 遊玩 → 空間壓縮 → Game Over → 分數上傳
                                  ↓
                          超越最高分？→ 成就感 → 再來一局
                          未超越？   → 遺憾感 → 再來一局（想改進）
```

### Long-Term Loop（非必要但可擴展）

- 排行榜競爭
- 解鎖新音效包
- 難度挑戰（Hard 模式最高分）

## 合成機制設計規範

### 規則定義模板

```markdown
## 機制：形狀合成（Shape Merge）

**目的**：核心動詞——兩個同級形狀碰撞時自動升級為下一等級
**玩家體驗**：「碰到了！」→ 視覺爆炸 + 分數跳出 → 滿足感
**輸入**：兩個相同等級的形狀發生物理碰撞
**輸出**：
  - 兩個原形狀消失
  - 在碰撞中點生成下一等級形狀
  - 得分 = 新形狀等級對應分數
  - 粒子效果 + 音效
**邊界條件**：
  - 最高等級（Lv.8）碰撞：不合成，視為普通碰撞
  - 多對同時碰撞：各自獨立處理
  - 合成後的新形狀立即可再觸發合成（連鎖）
**調節變數**：
  - 等級數量（Easy=7, Normal=8, Hard=9）
  - 各等級半徑（影響空間壓力）
  - 各等級得分（影響分數經濟）
```

### COMBO 機制規範

```
觸發條件：1200ms 窗口內連續合成
計數方式：comboCount 從 1 開始累加
得分加成：僅基礎分（不額外加分），COMBO 值為視覺/音效獎勵
重置條件：超過 1200ms 無新合成
```

## 難度平衡框架

| 變數 | Easy | Normal | Hard | 調節原則 |
|------|------|--------|------|---------|
| shapesSlice | 2（去掉前 2 種） | 1（去掉前 1 種） | 0（全部） | 縮減形狀總數 |
| 形狀總數 | 7 (Lv.0-6) | 8 (Lv.0-7) | 9 (Lv.0-8) | 等級越多→空間需求越大 |
| maxDropLevel | 3 | 3 | **4** | 可投放的最高等級 |
| 投放形狀池 | Lv.0~3 | Lv.0~3 | Lv.0~4 | Hard 多一個投放等級 |
| 容器寬度 | 476px | 476px | 476px | 固定不變 |
| Game Over 線 Y | 55px | 55px | 55px | 固定不變 |

**平衡原則**：
- 難度差異來自**形狀縮減**（探掉低等級）和**投放等級上限**，不來自物理參數
- 所有難度共用同一物理引擎設定
- 分數不因難度額外加成——排行榜按難度分組即可

## 玩家心理設計

### Pacing Chart 模板

```
時間      | 活動          | 張力    | 情緒
----------|-------------|--------|------------------
0:00-0:30 | 探索/學習     | Low    | 好奇
0:30-2:00 | 堆疊成長     | Medium | 信心建立
2:00-4:00 | 空間壓縮     | High   | 策略思考+緊張
4:00-6:00 | 大合成爽感   | Release| 成就感爆發
6:00-8:00 | 接近頂端     | Peak   | 焦慮/風險決策
8:00+     | Game Over   | Max    | 挫敗→「再來一局」
```

### 學習曲線設計原則

1. **0–15秒**：核心操作自明（點擊=投放）
2. **1–3局**：自然發現合成（而非文字教學）
3. **3–5局**：理解位置策略的重要性
4. **5+局**：開始利用 NEXT 預覽做規劃

## 數值設計 Checklist

- [ ] 每個等級的半徑增幅是否非線性遞增？
- [ ] 得分曲線是否讓高等級合成「值得追求」？
- [ ] 最大形狀是否佔容器面積 >10%？（空間壓力保證）
- [ ] COMBO 窗口時間是否讓「自然連鎖」可觸發？
- [ ] 難度差異是否只來自內容量，不來自物理感？
- [ ] 理論最高分是否在合理範圍？（不是無限也不是太低）

---

## 工作流程

1. **定義設計支柱** — 確認遊戲的核心體驗承諾
2. **撰寫核心循環** — 從玩家視角描述 moment-to-moment
3. **建立數值表** — 半徑/分數/難度的完整數據
4. **情感弧線** — 畫出 Pacing Chart
5. **邊界條件** — 定義每個機制的極端情況
6. **Playtest 驗證** — 與設計意圖對比實際體驗
