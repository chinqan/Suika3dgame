# 物理落下益智遊戲 — 空間與關卡設計師

**職能**：設計遊戲容器的空間架構、形狀進化系統、渲染層級、空間壓力曲線。

---

## When to Use

- 調整遊戲區域尺寸或邊界規格
- 設計/修改形狀進化表（等級、半徑、幾何形態）
- 分析空間佔用率與壓力曲線
- 設計新的難度模式或關卡變體
- 修改渲染層級（z-order）架構
- 調整 Game Over 判定機制

---

## 🚨 Critical Rules（不可違反）

- **難度參數必須精確對應 `DIFFICULTY_MAP` 結構** — 區分 `shapesSlice`（形狀減少）和 `maxDropLevel`（投放限制）
- **空間規格必須引用程式碼常數名** — 例如 `GAME_INSET_X = 35`、`FLOOR_AREA_H = 70`
- **渲染層級必須與 `layers.ts` 的實際 Container 名稱一致**

### 設計原則

- **寬高比 ≈ 0.73**：略寬於手機比例，堆疊有縱深感
- **Game Over 線 Y=55**：距頂端 55px，給予緊急時刻「最後一搏」空間
- **牆壁碰撞體**：不可見的剛體，形狀不會穿牆

## 形狀進化系統設計

### 進化表模板

```markdown
| Lv | 名稱 | 邊數 | 半徑 | 分數 | 顏色 | 容器佔比 |
|----|------|------|------|------|------|---------|
| 0  | 名稱 | N    | R px | S 分 | HEX  | ?.?%    |
```

### 半徑設計原則

1. **非線性遞增** — 低等級增幅小（+6~7px），高等級增幅大（+16~21px）
2. **Lv.5 為轉折點** — 之後空間壓力急遽增加
3. **最大形狀佔容器 >15%** — 保證空間緊張感
4. **2 個最大形狀 = 容器滿** — 理論上限制同時存在的最高等級數量

### 得分設計原則

1. **三角形數列為基底** — n(n+1)/2 提供自然遞增感
2. **最高等級可加碼** — 超出公式的額外獎勵（如 Lv.8 = 50 而非 45）
3. **低等級分數要低** — 讓「追高」成為有意義的策略選擇

### 形態設計原則

- **低等級 = 多邊形**（三角、正方、五邊形…）→ 堆疊不穩定
- **高等級 = 圓形**（大圓）→ 堆疊穩定但佔空間大
- **從尖銳到圓潤** — 視覺上暗示「進化 = 完美化」

## 渲染層級架構

```
Layer 7 (top)   : UI / HUD
Layer 6         : 浮動文字（分數、COMBO）
Layer 5         : 幽靈形狀（瞄準預覽）
Layer 4         : 活動形狀（玩家控制中）
Layer 3         : 已落地形狀
Layer 2         : 牆壁 + Game Over 線
Layer 1         : 背景格線 + 3D 地板
Layer 0 (bottom): 純色背景
```

**規則**：
- 層級不可跨越——幽靈永遠在形狀上方
- 粒子效果在 Layer 6（與浮動文字同級或獨立 sublayer）
- 格線光流在 Layer 1.5（背景之上、牆壁之下）

## Game Over 判定設計

```markdown
## 判定機制

**觸發條件**：形狀靜止後其頂部 Y 座標 < Game Over 線 Y
**精算公式**：shapeBody.position.y - shapeRadius < GAME_OVER_Y

**防止誤判**：
- 僅檢查「靜止」形狀（velocity < threshold）
- 合成瞬間不判定（新生成形狀有豁免期）
- 投落瞬間不判定（下落中的形狀排除）
```

## 空間壓力分析 Checklist

- [ ] Lv.8 佔容器面積百分比？（目標 >15%）
- [ ] 容器最多可容納幾個 Lv.8？（目標 ≤ 2）
- [ ] 平均局中同時存在多少個形狀？（測量空間飽和度）
- [ ] Game Over 線距離如何影響平均局時長？
- [ ] 不同難度下，初始 30 秒的空間壓力差異？

---

## 工作流程

1. **定義空間規格** — 容器尺寸、邊距、Game Over 線
2. **設計進化表** — 半徑、分數、形態、顏色的完整數據
3. **空間壓力分析** — 計算各等級的佔用比例
4. **渲染層級** — 確定所有圖層的 z-order
5. **邊界測試** — Game Over 誤判、牆壁穿透、底部堆疊
6. **難度變體** — 不同形狀數量對空間壓力的影響
